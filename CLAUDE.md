# RaviOS — Project Context

## Naming

Always write the product name as **RaviOS** (or **Ravi-OS** / **RAVI-OS** in all-caps contexts). Never write it as "RAVIOS" (no separator) or "ravios" — both are unreadable.



RaviOS is a playful, mysterious, terminal-inspired web portal designed for a sixth grader (Ravi / Agent RMP) who is beginning a short "vibe coding" course about building things on the internet.

This is NOT meant to feel like school, curriculum software, or a coding bootcamp.

## Emotional Goal

The emotional goal is:
- curiosity
- mystery
- delight
- empowerment
- experimentation
- "I can make real things on the internet"

The site should feel like:
- a secret system
- a hacker terminal
- an internet wizard academy
- an invitation into a hidden world

By the end of the course, Ravi should feel:
- "I can build things."
- "The internet is editable."
- "Coding is creative."
- "I can make weird stuff."
- "I am an internet wizard."

The ultimate goal is identity transformation: from consumer → creator. Not "learning programming."

## Tone & Visual Inspiration

- cinematic terminal UI
- retro-futuristic command line
- modern AI aesthetic
- playful but not childish
- slightly mysterious
- interactive and surprising

The landing page vibe loosely draws from the atmosphere of AI Daily Brief Terminal aesthetic:
- dark terminal environment
- animated boot sequence
- cinematic pacing
- clean typography
- subtle motion
- immersive command-line interactions

Do NOT copy any site directly — borrow the feeling.

## Core Concept

Ravi receives a link before the first class session. When he opens the site, it feels like he has been recruited into a secret internet-building program.

The homepage is primarily a fake interactive terminal.

Example opening sequence:

```
BOOTING RAVI-OS...

SECURE CONNECTION ESTABLISHED

WELCOME, AGENT RMP.

WE'VE BEEN EXPECTING YOU.

Your mission, should you choose to accept it:
Build things on the internet.

Current level: HUMAN
Target level: WIZARD

Type "help" to begin.
```

The site should create the feeling: "I'm entering a secret internet world."

## Technical Constraints

Ravi only has access to a Chromebook. Everything throughout the course should work:
- entirely in the browser
- without local installs
- beginner-friendly
- low-friction

The course stack is: GitHub, Vercel, v0.

The site itself:
- React + Vite or Next.js
- deployable on Vercel
- lightweight
- easy to extend later

## Missions

The site should present "missions" instead of lessons.

### Mission 0
Set up GitHub, Vercel, and v0. Framed as "unlocking internet builder tools."

### Mission 1
Build a personal homepage. Encourage favorite things, funny buttons, weird interactions, personality, experimentation. Goal: ship a real website quickly.

### Mission 2
Build an interactive project. Possible examples: joke generator, fortune teller, soundboard, mini game, battle simulator, weird internet toy. The exact project does not need to be decided yet.

## Terminal Features

The terminal should support:
- animated typing
- fake boot sequence
- command history
- fake system messages
- hidden commands
- easter eggs

Example commands: `help`, `mission`, `level`, `clear`, `family`, `joke`, `fortune`, `wizard`, `summon dragon`, `hack the mainframe`

Some commands should trigger animations, confetti, screen effects, dancing emoji sprites, or funny system responses. There should be hidden discoverable behavior. The site should reward curiosity.

## Design Principles

Prioritize:
- delight over polish
- responsiveness over complexity
- interactivity over information density
- mystery over explanation

Avoid:
- walls of text
- "educational platform" energy
- enterprise UI
- excessive onboarding
- over-explaining coding concepts

The site should feel alive.
